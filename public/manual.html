Partilha Pública com Doadores

Implementado

Como funciona

Admin clica "Partilhar"
Token gerado (12 hex chars)
URL copiado para clipboard
Doador abre no browser
✓
URL público: /album/[token] — sem login, abre em qualquer browser
✓
Página do doador: branding Chapateca, header gradient, info da actividade, grelha de fotos, observações
✓
Proxy seguro de fotos: /api/photo/[token]/[driveId] — verifica que a foto pertence ao álbum antes de servir (sem expor credenciais Google)
✓
Token persistente: uma vez gerado, o mesmo link funciona sempre. Fica guardado em Album.shareToken
✓
Middleware actualizado — rotas /album/* não precisam de autenticação
Ideal para WhatsApp: a Constance clica Partilhar, o URL é copiado automaticamente, e ela cola directamente numa mensagem. O doador abre uma página limpa sem precisar de criar conta.

Tarefas e Mensagens Internas

Implementado

Sistema de delegação de tarefas interno — admins atribuem tarefas a membros da equipa, que as recebem e confirmam execução.

Fluxo completo

Admin cria tarefa
/tarefas
Destinatário abre portal
Popup de notificação
"Recebi — vou tratar"
"Concluído" na inbox
✓
Criar tarefa — admins preenchem: Para (dropdown), Tarefa/Mensagem (obrigatório), Detalhe (opcional)
✓
Popup ao abrir o portal — aparece automaticamente se há tarefas não lidas (estado pending). Mostra quem enviou, o texto e botões de acção.
✓
3 estados: pending (não vista) → received (vista, em progresso) → done (concluída)
✓
Badge na sidebar — número de tarefas não lidas aparece ao lado de "Tarefas" na barra lateral
✓
Página /tarefas — Caixa de Entrada com todas as tarefas recebidas + (admins) secção Enviadas com estado de cada uma
✓
Botão "Concluído" — disponível na inbox quando a tarefa está em estado received
Email de notificação: ainda não implementado — fica para quando houver emails cadastrados na plataforma. A estrutura do código já suporta adicionar no servidor markTaskReceived.

Rotas de API

Implementado
✓
POST/api/upload
Recebe ficheiros + metadados, cria Album na DB, faz upload para Drive, cria FileLogs
✓
POST/api/share/[fileId]
Gera link público via Drive API (permissions.create)
✓
GET/api/drive/image/[driveId]
Proxy autenticado — serve foto do Drive para utilizadores com sessão activa
✓
GET/api/photo/[token]/[driveId]
Proxy público — verifica shareToken antes de servir a foto (para página de doadores)
✓
GET/api/logout
Apaga cookie de sessão e redirige para login

✓
